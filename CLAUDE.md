# にゃんポーカー プロジェクト

## 開発環境

- Python: uv で管理
- パッケージ追加: `uv add <package>`
- スクリプト実行: `uv run <script.py>`

## プロジェクト構成

- `docs/`: 仕様書
- `images/`: 猫の画像とmap.yml
- `mock/`: UX/UIモックファイル
- `tmp/`: 一時ファイル・分析用スクリプト

## 実装ルール

### UX/UIの一貫性確認

アプリ実装時は、必ず `mock/index.html` を参照してUXが壊れていないかを確認すること。

- 画面遷移、アニメーション、インタラクションはモックに準拠する
- デザイン（色、サイズ、レイアウト）はモックを正とする
- モックと異なる実装をする場合は、先にモックを更新してから実装する

## GitHub

### Issue登録ルール

Issueを登録する際は以下のルールに従うこと。

#### タイトル

- 簡潔かつ具体的に記述する
- プレフィックスを付ける: `[Bug]`, `[Feature]`, `[Improvement]`, `[Docs]` など

#### 本文テンプレート

**Bugの場合:**
```markdown
## 概要
（バグの簡潔な説明）

## 再現手順
1.
2.
3.

## 期待する動作
（本来どうあるべきか）

## 実際の動作
（現在何が起きているか）

## 環境
- OS:
- Python:
- その他関連情報:
```

**Featureの場合:**
```markdown
## 概要
（機能の簡潔な説明）

## 背景・目的
（なぜこの機能が必要か）

## 要件
- [ ] 要件1
- [ ] 要件2

## 参考
（関連するドキュメント、モック、画像など）
```

#### ラベル

適切なラベルを付与する:
- `bug`: バグ報告
- `enhancement`: 機能追加・改善
- `documentation`: ドキュメント関連
- `question`: 質問・相談
- `priority:high/medium/low`: 優先度
